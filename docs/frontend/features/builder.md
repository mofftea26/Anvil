# Workout Builder

## Status

Implemented (writes to both normalized tables and `workouts.state` JSONB; `state` is the source of truth per ADR-005).

## Purpose

Trainers use the workout builder to author standalone workouts and per-day workouts inside a program template. The builder presents a horizontal carousel of "series" pages (warmup / main / cooldown / etc.), each containing exercises with editable set prescriptions (reps, weight, rest, set type, tempo, RPE).

## User Flow

1. Trainer enters the builder from one of three entry points:
   - `app/(trainer)/library/workout-builder/new.tsx` → blank workout (`mode: "new"`).
   - `app/(trainer)/library/workout-builder/[workoutId].tsx` → editing an existing workout (`mode: "edit"`).
   - From a program-template phase day → carries `pendingProgramDayAttachment` so on save the workout is wired into that day.
2. The builder loads the workout's `state` JSONB (or starts empty), shows a carousel of series pages.
3. The trainer picks exercises (`/library/workout-builder/exercise-picker.tsx` → `ExercisePickerScreen`), opens `SetsEditorSheet` to edit prescriptions, taps the title to rename, etc.
4. Saving calls `saveWorkout` (in `features/library/api/workouts.api.ts` / `features/builder/api/workouts.api.ts`) — writes `workouts.state` and the legacy normalized children.
5. Exercise detail (`app/(trainer)/library/workout-builder/exercise/[exerciseId].tsx` → `ExerciseDetailScreen`) is reached from any exercise card to view media (image / video) and metadata.

## Components

- `SeriesPage` + `AddSeriesCard` (`components/SeriesPage.tsx`) — one carousel page per series.
- `SetsEditorSheet` (`components/SetsEditorSheet.tsx`) — bottom-sheet editor for an exercise's set prescriptions.
- `SetTypePickerSheet` (`components/SetTypePickerSheet.tsx`) — choose a set type from the dictionary.
- `ExerciseCard` (`components/ExerciseCard.tsx`) — exercise summary used by series and picker.
- `VideoPlayerModal` (`components/VideoPlayerModal.tsx`) — full-screen exercise video preview.

## Hooks

- `useWorkoutEditor` — owns the in-memory `state` (series, exercises, prescriptions), undo flags, dirty tracking, and persistence wiring.
- `useVideoThumbnail` — generates a thumbnail for an exercise video URL via `expo-video-thumbnails`.
- `useSetTypesDictionary` (from `features/library/`) — read-only set-type catalog.
- `useFocusEffect` is used to consume pending picks from `exercisePickerBridge` and `programDayAttachmentBridge` when the screen regains focus.

## State Management

- Per-screen local state via `useWorkoutEditor` (no Redux slice).
- Cross-screen handoff via two in-memory bridges:
  - `exercisePickerBridge` — picker writes the chosen exercise; builder reads on focus.
  - `programDayAttachmentBridge` (in `features/library/utils/`) — wires a fresh save into a program day.
- Reads of set-type dictionary and program data go through their respective feature hooks.

## API / Supabase Dependencies

- `workouts` (`workouts.state` JSONB is canonical per ADR-005) and the normalized `workoutSeriesBlocks` / `workoutSeriesExercises` / `workoutSetPrescriptions` (still written for legacy reads — see `technical-debt.md`).
- `exercises` for the picker.
- `setTypes` for set-type catalog.
- `programPhases` / `programPhaseDays` when wiring a built workout into a program day.
- RPC: `anvil_update_workout_state` (planned consolidation per ADR-005).

## Validation Rules

- Title required; trimmed.
- Each series must have a non-empty label (auto-generated by `getNextSeriesLabel`).
- Each set prescription must have non-negative `targetReps`, `targetWeightKg`, `restSec`.
- No duplicate exercise within the same series (UI prevents).

## UI / UX Rules

- Dark-only theme; uses `useTheme()` for all colors.
- Horizontal `FlatList` carousel for series pages with paginated snap.
- `SetsEditorSheet` and `SetTypePickerSheet` use `BottomSheetPicker` from `@/shared/ui`.
- Long-press exercise card to delete; tap to open detail.
- Builder screens (builder, picker, and exercise detail) use shared app-wide horizontal gutter spacing via `getScreenHorizontalPadding`.

## iOS + Android Notes

- `FlatList` carousel uses `pagingEnabled` + `decelerationRate="fast"`; tested for parity.
- Video thumbnails are generated lazily and cached in module-level `Map`.
- Keyboard avoidance handled by sheets; main screen does not need `KeyboardScreen` because edits happen in sheets.

## SOLID / Architecture Notes

- Editor logic lives in `useWorkoutEditor` (hook-owns-side-effects).
- Save logic is delegated to `features/builder/api/workouts.api.ts` (presentation layer doesn't call Supabase directly).
- The picker feeds the builder via a bridge utility instead of nested navigation params, keeping the builder agnostic of who navigated to it.

## Performance Notes

- Carousel renders only the in-view series page; off-screen pages keep their `useState` but skip heavy work.
- Set rows are presentation-only and re-render on prescription change (acceptable; small sets).
- Video thumbnails are generated once and memoized.
